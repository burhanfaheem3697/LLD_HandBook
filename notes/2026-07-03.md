## 🧩 Key Concepts

### 1. Static vs. Dynamic Dependencies

| Type | Description |
|---|---|
| **Static Dependencies** | Compile-time relationships showing how modules/classes structurally reference each other (`import` statements, class declarations). If Class A statically references Class B, B must exist at compile time. |
| **Dynamic Execution Dependencies** | Occur purely at runtime — object interactions, message passing, conditional paths, polymorphic behavior over time. |

### 2. Physical File Mappings & The Classpath

- **Namespace Resolution** — `javac`/`java` map package declarations (`package com.handbook.core;`) directly to physical directories (`com/handbook/core/`).
- **The Root Rule** — Multi-package projects must be compiled from the root source directory so the compiler can trace relative package paths.
- **Compilation Path vs. Runtime Namespace**:

  | Tool | Targets | Separator | Extension |
  |---|---|---|---|
  | `javac` | Physical file paths | `/` or `\` | `.java` (required) |
  | `java` | Compiled classes in JVM namespace | `.` | none |

### 3. Dynamic Class Loading & Fault Tolerance

- **Lazy Loading** — Java links classes on-demand at runtime, rather than binding everything into one binary at build time.
- **Soft Probing** — `Class.forName()` lets a program check whether an optional dependency (e.g., a JDBC driver) exists on the classpath. A missing class throws a checked `ClassNotFoundException`, which can be caught to gracefully disable a feature instead of crashing.

## 🚀 Step 4 — Hands-on Lab: Application Bootstrapper & Class Loader Validator

The **Bootstrapper pattern** acts as an environmental gatekeeper — running validation checks and initializing configs before the rest of the app starts.

### Control Flow Under Invariant Failures

- **Short-Circuiting Behavior** — In Test Case 1, a missing `"environment"` config causes `ConfigLoader.getRequired()` to immediately throw an `IllegalArgumentException`.
- **Execution Interruption** — The exception halts `boot()` on the spot, skipping remaining routines (like `isClassAvailable()`) and jumping straight to the caller's `catch` block — preventing the app from half-booting into an unsafe state.

## 🏗️ Real-world Connections & Architectural Visualizations

### UML Sequence Diagrams vs. Structural Class Diagrams

| Diagram Type | Models |
|---|---|
| **UML Class Diagrams** | Static, timeless blueprint — *what* exists (classes, attributes, operations) and how they link (inheritance, composition, association). |
| **UML Sequence Diagrams** | Dynamic behavior over a vertical timeline — *how* objects interact, pass messages, activate loops, and branch via alt/opt fragments. |

### Production Directory Splitting

Separating `src/main/java/` (production) from `src/test/java/` (testing) enables clean classpath management — keeping heavy test frameworks (JUnit, Mockito) out of production artifacts.

## 💬 Engineering Q&A Log

**Q: Why does the compiler automatically build `Part.java` when I only tell it to compile `Engine.java`?**
> `javac` handles transient dependencies automatically. When it hits the `import` for `Part`, it searches the classpath for a compiled `Part.class` or raw `Part.java`, and compiles the source file behind the scenes if found.

**Q: Why is `ClassNotFoundException` checked instead of unchecked?**
> An unconfigured or shifting classpath is an external, environmental factor the compiler can't guarantee against. A checked exception forces developers to write a recovery/fallback strategy so missing classes don't crash the app unhandled.

**Q: What happens to `assert` checks without the `-ea` flag?**
> The JVM strips assertion instructions entirely at runtime — treating them like comments. No invariant checking occurs, and no `AssertionError` is ever raised.
