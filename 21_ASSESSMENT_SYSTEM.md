# 21_ASSESSMENT_SYSTEM

Every phase assessment contains the same categories.

## Assessment Suite

| Category | Evidence |
|---|---|
| Knowledge Quiz | short answers, definitions, trade-off explanations |
| Machine Coding | working Java implementation under constraints |
| Code Review | review another solution or prior version |
| Architecture Review | diagrams, ADRs, quality attributes |
| Refactoring Exercise | before/after behavior-preserving improvement |
| Design Review | responsibility assignment and alternatives |
| Portfolio Review | README, docs, tests, diagrams, CI |
| Self Evaluation | written reflection against mastery checklist |
| Mastery Checklist | observable capabilities |

## Rubric

| Level | Description |
|---|---|
| 1 | Can follow examples but cannot adapt them reliably |
| 2 | Can implement narrow solutions with guidance |
| 3 | Can design maintainable solutions independently |
| 4 | Can evaluate trade-offs and guide others |
| 5 | Can design reusable frameworks and production architecture |

## Mastery Evidence

By the end of the handbook, the learner must demonstrate:

- Java fluency
- Object modeling skill
- Refactoring discipline
- Pattern judgment
- Architecture reasoning
- Framework API design
- JVM and systems literacy
- Concurrency and networking competence
- Persistence and distributed systems foundations
- Event-driven design
- Production engineering practice
- Open-source reading and contribution skill
- Capstone-level integration
